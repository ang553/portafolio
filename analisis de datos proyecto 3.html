Diccionario de datos¶

Hay cinco tablas en el conjunto de datos, y tendrs que usarlas todas para hacer el preprocesamiento de datos y el anlisis exploratorio de datos. A continuacin se muestra un diccionario de datos que enumera las columnas de cada tabla y describe los datos que contienen.

  • instacart_orders.csv: cada fila corresponde a un pedido en la aplicacin Instacart.
    • 'order_id': nmero de ID que identifica de manera nica cada pedido.
    • 'user_id': nmero de ID que identifica de manera nica la cuenta de cada cliente.
    • 'order_number': el nmero de veces que este cliente ha hecho un pedido.
    • 'order_dow': da de la semana en que se hizo el pedido (0 si es domingo).
    • 'order_hour_of_day': hora del da en que se hizo el pedido.
    • 'days_since_prior_order': nmero de das transcurridos desde que este cliente hizo su pedido anterior.
  • products.csv: cada fila corresponde a un producto nico que pueden comprar los clientes.
    • 'product_id': nmero ID que identifica de manera nica cada producto.
    • 'product_name': nombre del producto.
    • 'aisle_id': nmero ID que identifica de manera nica cada categora de pasillo de vveres.
    • 'department_id': nmero ID que identifica de manera nica cada departamento de vveres.
  • order_products.csv: cada fila corresponde a un artculo pedido en un pedido.
    • 'order_id': nmero de ID que identifica de manera nica cada pedido.
    • 'product_id': nmero ID que identifica de manera nica cada producto.
    • 'add_to_cart_order': el orden secuencial en el que se aadi cada artculo en el carrito.
    • 'reordered': 0 si el cliente nunca ha pedido este producto antes, 1 si lo ha pedido.
  • aisles.csv
    • 'aisle_id': nmero ID que identifica de manera nica cada categora de pasillo de vveres.
    • 'aisle': nombre del pasillo.
  • departments.csv
    • 'department_id': nmero ID que identifica de manera nica cada departamento de vveres.
    • 'department': nombre del departamento.
